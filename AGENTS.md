## The Unified Triple-Core Design & QA Framework

Every code modification must be evaluated and verified against the Triple-Core pillars described in detail in **[MOCKMAXXING_BIBLE.md](file:///c:/Users/loubr/.gemini/antigravity/scratch/snap-count/MOCKMAXXING_BIBLE.md#3-pillar-2-the-triple-core-design--qa-framework)**:

### 1. Apple Human Interface Guidelines (HIG) (Aesthetic & Motion)
* **Directives**: Apply elegant, custom typography, high-fidelity visual depth, fluid physics-based spring animations (no rigid snaps), and native system components with light haptic triggers. Keep interactions tactile and responsive with rounded press highlights.

### 2. Google Material Design 3 (M3) (Containment & States)
* **Directives**: Structure layouts using strict container boundaries. Ensure clear physical containment for interactive components (menus, cards, popovers, sheets). Define clear interaction states (default, hovered, focused, pressed, disabled) with proper elevation shadows.

### 3. WCAG 2.2 AAA Accessibility Standards (Legibility & Backdrops)
* **Directives**: All critical text overlay systems must guarantee a minimum color contrast ratio of **7:1** against their background surfaces (large text >= 4.5:1). Enforce solid black text on all energetic Champagne Bronze (`#bea98e`) surfaces.
* **Opaque Backdrop Mandate**: Sticky headers, bottom tab navigation bars, and dropdown popovers overlaying dynamic scrolling elements or high-density grids **must utilize a 100% opaque solid background color** (e.g., `#2c2c2c`) to completely prevent scrolled text bleed and contrast violations.

### 4. The Starbucks Inspiration Mandate (Memory & Anchor Rule)
* **Directives**: When the user refers to, details, or provides media from **Starbucks**, this is **strictly** a request to analyze, design, and draft **MockMaxxing UI layouts, features, and assets** through the lens of that Starbucks UX/UI/interaction inspiration.
* **Rule**: Do **NOT** change brand colors to Starbucks green/brown/gold, and do **NOT** adopt Starbucks terminology or trademarks. All designs must adhere strictly to the MockMaxxing corporate palette (Obsidian Black `#0c0c0c`, Elegant Graphite `#2c2c2c`, Sleek Slate `#4a4a4a`, Champagne Bronze `#bea98e`).
* **Rule**: Translate Starbucks' high-fidelity structural paradigms (such as segmented capsule switchers, interactive rewards cards, swiping cards/badges, sticky scrolling elements, sheet slides) into elite MockMaxxing assets and fantasy sports terminology.

### 5. Mandatory Dark Mode Verification Mandate
* **Directives**: Any time any modifications are made to the codebase, screens, layouts, or individual styling blocks, developers and agents MUST execute a rigorous verification pass specifically within **Dark Mode** to ensure absolute visual harmony, zero invisible text/UI elements, and 100% WCAG AAA contrast compliance against Obsidian Graphite backdrops.
